ezcDocumentPdfTextBoxRenderer

Extends \ezcDocumentPdfBlockRenderer

Renders a single text box

Tries to render a single text box into the available space, and aborts if not possible.

Implements the basic methods for tokenizing the text, style based text fitting into lines, etc. Should be extended by all classes implementing more specific text rendering algorithms, since those base methods are implemented generally enough to be reused.

Force the splitting of a word, which did not fit in a line alone and could not be splitted using the hyphenator. We just search for the maximum word part length which fits the available space.

Could be improved to use a binary search on the word length, but this shouldn't happen too often anyways.

Tokenize the input string

tokenize(\ezcDocumentLocateableDomElement $element, \ezcDocumentPdfTokenizer $tokenizer, boolean $recursed = false) : array

For proper word wrapping in the paragraph the strng needs to be tokenized, while each token has to maintain its stack of assigned formats.

This method should return an array of tokens, also maintaining the included whitespace characters, each associated with its markup elements.
